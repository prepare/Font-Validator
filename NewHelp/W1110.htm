Message: W1110

glyf: Intersecting components of composite glyph

The glyph indicated in the validation log contains contours of glyph components that intersect (overlap). This type of glyph design introduces some ambiguity in terms of defining what is the 'inside' and 'outside' of a glyph shape, and may cause a rasterizer to reverse the overlapped portion. This problem is generally seen on older printers. Although Microsoft discourages this type of glyph design, for legacy reasons this overlap is allowed for composite glyphs.

 

The current implementation of the rasterizer as well as many legacy versions handle this exception and ensures that the overlap between components is filled.

Please note that for consistency and predictability across platforms and implementation of the rasterizer, composite glyphs with overlapping contours should be redrawn altogether as a single glyph.

For more information, see the glyf table specification.
